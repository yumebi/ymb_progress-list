# 進行状況リスト (progress-list)

案件の進行状況を管理し、メール等に貼り付けられるテキスト一覧を出力するTauriアプリ。

```
■サンプルカテゴリA
※26/06/08　サンプル案件1　月次修正－確認中、発注書：有
・26/06/18　サンプル案件2－進行中、素材マチ（50%）
・28/02/??　サンプル案件3
```

## 案件行の項目

| 項目 | 入力方法 | 出力 |
|---|---|---|
| 済(記号) | プルダウン(マスタ設定で可変。・※✗など) | 行頭にそのまま |
| 日付 | 年・月・日それぞれ個別にプルダウン(各「未定」可)、または📅ボタンからカレンダーで一括選択 | `YY/MM/DD`(未定の項目は `??`) |
| 案件名 | テキスト | そのまま |
| ステータス | プルダウン(マスタ設定で可変) | `－確認中` など |
| 進捗率 | スライダー(ドラッグ)or数値 0-100% | 1%以上で `（50%）` をステータス直後に付与 |
| 発注書 | プルダウン(マスタ設定で可変。有/無/済。デフォルトは「（なし）」) | 「（なし）」以外を設定時のみ末尾に `発注書：有` |

- カテゴリ(`■サンプルカテゴリA` など)は「＋カテゴリ追加」で自由に追加・改名・並べ替え・削除可。
- 各カテゴリ内は日付昇順に自動ソート(日付未定は先頭。年だけ・年月だけの指定も可)。
- プルダウンの選択肢(済・ステータス・発注書)はツールバーの **マスタ設定** で追加・削除・並べ替え可。
- 画面右上で **ダーク/ライト** 切り替え(初期値はダーク)。
- 各列の幅・出力プレビューの幅は、列見出しや境界のハンドルをドラッグして変更可能(設定は自動保存)。
- **コピー** でクリップボードへ、**.txt保存** でファイル出力。
- データは `%APPDATA%\com.yumebi.progress-list\data.json` に自動保存(変更の500ms後)。
- 旧形式のデータは起動時に自動変換される([storage.ts](src/storage.ts) の `migrate`)。

## 開発

```bash
npm install
npm run tauri dev    # 開発起動
npm run tauri build  # リリースビルド(src-tauri/target/release/)
```

主要ファイル:

- `src/format.ts` — ソート・出力テキスト生成
- `src/storage.ts` — tauri-plugin-store による永続化+旧データ変換(ブラウザ実行時はlocalStorage)
- `src/seed.ts` — 初回起動時の初期データ
- `src/App.tsx` — UI全体(カテゴリ/案件編集・マスタ設定・プレビュー・テーマ切替)
